# Плотина — Облачная IaaS-платформа

## Презентация продукта для потенциального заказчика

---

## 1. О продукте

**Плотина** — это современная платформа Infrastructure-as-a-Service (IaaS), разработанная командой LSBU в рамках хакатона MTS x BSU. Платформа позволяет пользователям создавать, управлять и масштабировать виртуальные серверы (VPS) через удобный веб-интерфейс с интегрированным AI-ассистентом.

**Слоган:** *«Стройте на Плотине»*

---

## 2. Целевая аудитория

### Основные сегменты:
- **Малый и средний бизнес (МСБ)** — компании, которым нужна облачная инфраструктура без сложной настройки и больших вложений в собственное оборудование
- **Стартапы и разработчики** — быстрое развертывание серверов для MVP и прототипов с AI-подсказками по оптимальной конфигурации
- **Образовательные учреждения** — предоставление вычислительных ресурсов студентам и исследователям
- **Внутренние подразделения MTS** — приватное облако для внутренних сервисов и тестовых сред

### Почему наш продукт необходим:
1. **Простота** — интуитивный интерфейс, не требующий глубоких знаний системного администрирования
2. **AI-помощник** — встроенный чат-бот помогает подобрать конфигурацию и отвечает на вопросы по облачной инфраструктуре
3. **Прозрачное ценообразование** — динамические цены за ресурсы (CPU, RAM, диск) с калькулятором стоимости в реальном времени
4. **Маркетплейс шаблонов** — возможность делиться готовыми конфигурациями серверов через снапшоты

## 3. Ключевые функции

| Функция | Описание |
|---------|----------|
| **Управление VPS** | Создание, запуск, остановка, удаление серверов через веб-интерфейс |
| **Веб-терминал** | Доступ к серверу через браузер по WebSocket |
| **Виртуальные сети** | Создание приватных и публичных сетей, подключение контейнеров |
| **Маркетплейс снапшотов** | Публикация и развертывание готовых серверных конфигураций |
| **AI-рекомендации** | Подбор оптимальной конфигурации оборудования на основе требований пользователя |
| **AI-консультант** | Чат-бот для вопросов по облачной инфраструктуре с памятью диалога |
| **Мониторинг метрик** | Отслеживание CPU, RAM, диска и сетевого трафика в реальном времени |
| **Админ-панель** | Управление пользователями, нодами, контейнерами и лимитами |
| **Порт-маппинг** | Настройка перенаправления портов для доступа к сервисам |
| **Динамическое ценообразование** | Цены привязаны к конкретной ноде и настраиваются администратором |

## 4. Архитектура

### 4.1 Общая схема

Платформа построена на **микросервисной архитектуре** из 7 Docker-контейнеров:

```
┌─────────────────────────────────────────────────────────────────┐
│                        Frontend (React)                         │
│                   React 19 + TypeScript + Vite                  │
└────────────────────────────┬────────────────────────────────────┘
                             │ HTTP / WebSocket
┌────────────────────────────▼────────────────────────────────────┐
│                     Go API Server (net/http)                    │
│            REST API + WebSocket + JWT аутентификация             │
├──────────┬──────────┬───────────┬───────────┬───────────────────┤
│  gRPC    │   SQL    │   AMQP    │   AMQP    │     WebSocket     │
│          │          │           │           │                   │
▼          ▼          ▼           ▼           ▼
┌───────┐ ┌────────┐ ┌─────────┐ ┌─────────┐ ┌─────────────────┐
│  LXD  │ │Postgres│ │RabbitMQ │ │  Redis   │ │  Веб-терминал   │
│ Nodes │ │  16    │ │  4.2.4  │ │  7.0     │ │  (контейнеры)   │
└───────┘ └────────┘ └────┬────┘ └─────────┘ └─────────────────┘
                          │
              ┌───────────┴───────────┐
              ▼                       ▼
    ┌──────────────────┐   ┌──────────────────┐
    │  Recommendation  │   │   AI Consult     │
    │    Service        │   │    Service       │
    │  (Python 3.11)   │   │  (Python 3.11)   │
    │  Gemini 2.0 Flash│   │  Gemini 2.0 Flash│
    └──────────────────┘   └──────────────────┘
```

### 4.2 Компоненты инфраструктуры

#### PostgreSQL 16 — Основная база данных
- Хранение пользователей, контейнеров, нод, сетей, снапшотов, порт-маппингов
- Параметризованные SQL-запросы для защиты от SQL-инъекций
- Драйвер pgx/v5 для высокой производительности

#### RabbitMQ 4.2.4 — Брокер сообщений
- **Паттерн RPC:** Go API отправляет запрос в очередь → Python-воркер обрабатывает → результат возвращается через callback-очередь с correlation ID
- **Две очереди:**
  - `hardware_requests` — запросы на подбор конфигурации оборудования
  - `chat_requests` — сообщения AI-чата
- **Зачем нужен:** Развязывает синхронный Go API и асинхронные Python AI-сервисы, обеспечивает надёжную доставку сообщений и масштабирование воркеров
- **Преимущества:** Гарантия доставки, буферизация при пиковых нагрузках, возможность горизонтального масштабирования воркеров

#### Redis 7.0 — Кеш и хранилище сессий
- Хранение истории чата по ключу `chat:{user_id}`
- Ограничение: последние 20 сообщений, TTL — 24 часа
- При запросе к AI отправляются последние 5 сообщений для контекста
- **Зачем нужен:** Быстрое in-memory хранилище для данных с ограниченным сроком жизни, не нагружает основную БД

#### LXD — Контейнеризация
- Контейнеры Linux (LXC) для изоляции VPS
- Хранилище: btrfs (20 ГБ пул)
- Сеть: bridge-интерфейс (lxdbr0)
- Поддержка вложенных контейнеров (security.nesting)
- **Go Agent** на каждой ноде — gRPC-сервер для управления контейнерами, iptables-правилами и сбора метрик

### 4.3 Стек технологий

| Слой | Технологии |
|------|-----------|
| **Frontend** | React 19, TypeScript 5.9, Vite 7.3, Tailwind CSS 4.2, Axios |
| **Backend API** | Go (net/http, Go 1.22+ ServeMux), gorilla/websocket, JWT |
| **AI-сервисы** | Python 3.11, Google Gemini 2.0 Flash, asyncio, aio-pika |
| **База данных** | PostgreSQL 16 (pgx/v5) |
| **Очереди** | RabbitMQ 4.2.4 (AMQP 0.9.1) |
| **Кеш** | Redis 7.0 |
| **Контейнеризация** | LXD/LXC, Docker Compose |
| **Связь с нодами** | gRPC + Protocol Buffers |
| **Управление портами** | iptables NAT |

## 5. Преимущества

### Технические:
- ✅ **Микросервисная архитектура** — каждый компонент независим и масштабируем
- ✅ **Асинхронная обработка AI** — RabbitMQ развязывает API и AI-сервисы, исключая блокировки
- ✅ **Двойной AI-функционал** — рекомендации по конфигурации + чат-консультант
- ✅ **Реальная виртуализация** — LXD-контейнеры с полной изоляцией ресурсов
- ✅ **gRPC для управления нодами** — высокопроизводительный бинарный протокол
- ✅ **Мониторинг в реальном времени** — стриминг метрик через gRPC и WebSocket
- ✅ **Безопасность** — JWT-аутентификация, параметризованные запросы, проверка владения ресурсами
- ✅ **Веб-терминал** — доступ к серверу прямо из браузера без SSH-клиента

### Продуктовые:
- ✅ **Маркетплейс** — экосистема готовых конфигураций ускоряет развертывание
- ✅ **AI на борту** — пользователь получает рекомендации без экспертных знаний
- ✅ **Динамические цены** — администратор настраивает стоимость для каждой ноды
- ✅ **Адаптивный интерфейс** — работает на мобильных устройствах и десктопе
- ✅ **Полностью на русском языке** — локализованный интерфейс для целевого рынка

## 6. Ограничения и зоны роста

### Текущие ограничения:
- ⚠️ **Отсутствие автоскейлинга** — масштабирование ресурсов контейнеров вручную
- ⚠️ **Нет биллинговой интеграции** — баланс пополняется виртуально, без подключения к платежным системам
- ⚠️ **Одна зона доступности** — нет георепликации и отказоустойчивости на уровне дата-центров
- ⚠️ **Нет rate-limiting** — отсутствует ограничение запросов на уровне API
- ⚠️ **Ограниченный мониторинг** — базовые метрики CPU/RAM/диск без алертов и дашбордов
- ⚠️ **Нет резервного копирования** — отсутствует автоматическое бэкапирование данных пользователей

### Планы развития:
- 🔮 Интеграция с платежными системами (банковские карты, SBP)
- 🔮 Kubernetes-кластеры как сервис (KaaS)
- 🔮 Автоскейлинг на основе метрик
- 🔮 CI/CD пайплайны для пользователей
- 🔮 Балансировка нагрузки и георепликация
- 🔮 Подробные дашборды мониторинга (Grafana-интеграция)

## 8. Безопасность

- **Аутентификация:** JWT с access- и refresh-токенами
- **Авторизация:** Middleware проверяет права доступа на каждом эндпоинте
- **Изоляция данных:** Проверка владения ресурсами (контейнеры, сети, снапшоты)
- **SQL-инъекции:** Параметризованные запросы во всех операциях с БД
- **Валидация входных данных:** Серверная + клиентская валидация регистрации, баланса
- **Контейнерная изоляция:** LXD обеспечивает системную изоляцию между VPS

## 9. Резюме

**Плотина** — это полнофункциональная IaaS-платформа, которая выделяется среди конкурентов благодаря:

1. **Интеграции AI** — два AI-сервиса (рекомендации + консультант) на базе Google Gemini снижают порог входа для пользователей
2. **Современной архитектуре** — микросервисы, брокер сообщений, кеширование, gRPC — обеспечивают масштабируемость и надёжность
3. **Полноте функционала** — от создания VPS до маркетплейса шаблонов и мониторинга в одном интерфейсе
4. **Готовности к масштабированию** — Docker Compose позволяет быстро перейти на Kubernetes при росте нагрузки
